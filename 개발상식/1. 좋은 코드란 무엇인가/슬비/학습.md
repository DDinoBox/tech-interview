# 좋은 코드란 무엇인가?

### **좋은 코드란?**

> 일관성 있는 코드를 작성함으로써 읽기 쉽고, 중복이 없고, 테스트를 하기 좋은 코드

- 읽기 쉬운 코드
- 중복이 없는 코드
- 테스트를 하기 좋은 코드

</br>

### 1. 읽기 쉬운 코드

최소한의 가독성을 보장하는 방법은 일관성 있는 코드를 작성하는 것이다. 또, 읽기 쉬운 코드는 단순히 코드를 의미하는 것이 아니라 이해하는 것까지 포함되기 때문에 읽기 쉬울수록 좋은 코드라고 할 수 있다.

1-1. 일관성이 있는 네이밍을 작성한다.

- 변수명 뿐만 아니라 함수명에서도 규칙이 있으면 좋다.
- 리액트 훅스에서는 prefix - action - target 으로 나타낸다. (ex. onClickButton, handleButtonClick)

1-2. 일관성이 있는 들여쓰기를 한다.

- 파이썬은 정확히 들여쓰기가 지켜져야 정상 작동을 하지만, JS의 경우 그렇지 않으므로 팀내 일관성 있는 들여쓰기가 필요하다.

1-3. Directory 구조

- 일관된 디렉터리 구조는 전체적인 구조를 파악하는데 큰 도움이 되고 컴포넌트 간의 관계를 파악하는데에도 도움이 된다.

</br>

### 2. 중복이 제거된 코드

중복 코드는 재사용성이 떨어지며, 동일한 코드를 반복해서 사용하는 것은 노동력이 많이 소모된다.
중복이 제거된 코드는 리팩토링 기법 중 가장 효과가 좋으며 유지보수, 장애 처리 비용에 도움이 된다. 또, 문서화 하기에도 용이하고 코드의 품질을 높이는 확실한 방법이다.

- 함수로 분리하여 로직을 재사용할 수 있게 만들 수 있다.

</br>

### 3. 테스트를 하기 좋은 코드

테스트 코드를 작성하면 개발 과정에서 문제를 미리 발견하고, 빠른 시간 내 코드의 동작 방식과 결과를 확인할 수 있다. 의도한 대로 동작되는 것
더불어 좋은 테스트 코드를 연습하다 보면 자연스럽게 좋은 코드가 만들어진다.

- 같은 입력에 항상 같은 결과를 반환하는 코드
